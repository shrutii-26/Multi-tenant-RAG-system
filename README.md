# 🧠 Multi-Tenant RAG System

A Retrieval-Augmented Generation (RAG) system that lets users upload PDF documents and query them using natural language. Each upload creates an isolated knowledge base, making it multi-tenant by design.

**Live Demo**: [your-app.streamlit.app] https://multi-tenant-rag-system-lbgkmgi2dnz7tjzt3dkcch.streamlit.app/

---

## What it does

- Upload one or more PDF files
- The system chunks, embeds, and indexes the content using FAISS
- Ask any question in natural language
- Get answers grounded in your documents, generated by Groq's LLaMA model

---

## Tech Stack

| Layer        | Technology                                |
| ------------ | ----------------------------------------- |
| Frontend     | Streamlit                                 |
| Embeddings   | SentenceTransformers (`all-MiniLM-L6-v2`) |
| Vector Store | FAISS (local)                             |
| LLM          | Groq API (LLaMA 3.1)                      |
| PDF Parsing  | PyPDF2                                    |
| Deployment   | Streamlit Cloud                           |

## Project Structure

```
├── app.py               # Streamlit frontend
├── ingestion.py         # PDF parsing, chunking, FAISS index building
├── retrieval.py         # Semantic search over FAISS index
├── generator.py         # Answer generation via Groq LLM
├── embedding_model.py   # SentenceTransformer loader (singleton)
├── embeddings.py        # Embedding helper
├── config.py            # Model name, chunk size, overlap, top-k
├── indexes/             # FAISS indexes stored per knowledge base (ephemeral)
└── requirements.txt
```

---

## How it works

1. **Ingestion**: Uploaded PDFs are parsed and split into overlapping text chunks (300 tokens, 60 token overlap). Each chunk is embedded using `all-MiniLM-L6-v2` and stored in a FAISS flat L2 index.

2. **Retrieval**: The user's question is embedded using the same model. FAISS retrieves the top-k most semantically similar chunks.

3. **Generation**: Retrieved chunks are passed as context to Groq's LLaMA model, which generates a grounded answer.

## Running Locally

**1. Clone the repo**

```bash
git clone https://github.com/shrutii-26/Multi-tenant-RAG-system
cd shrutii-26/Multi-tenant-RAG-system
```

**2. Create a virtual environment**

```bash
python -m venv venv
venv\Scripts\activate      # Windows
source venv/bin/activate   # Mac/Linux
```

**3. Install dependencies**

```bash
pip install -r requirements.txt
```

**4. Set up environment variables**

Create a `.env` file at the root:

```
GROQ_API_KEY=your_groq_api_key_here
```

**5. Run the app**

```bash
streamlit run app.py
```

Opens at `http://localhost:8501`

---

## Deploying on Streamlit Cloud

1. Push this repo to GitHub
2. Go to [share.streamlit.io](https://share.streamlit.io) and connect your repo
3. Set main file path to `app.py`
4. Under **Advanced settings → Secrets**, add:

```
GROQ_API_KEY = "your_groq_api_key_here"
```

5. Deploy

> **Note**: FAISS indexes are stored on disk and are ephemeral on Streamlit Cloud. Users will need to re-upload their PDFs after an app restart. This is expected behavior for a stateless deployment.

## Configuration

All tunable parameters are in `config.py`:

```python
MODEL_NAME = "llama-3.1-8b-instant"   # Groq model
CHUNK_SIZE = 300                        # Characters per chunk
OVERLAP    = 60                         # Overlap between chunks
TOP_K      = 4                          # Chunks retrieved per query
```
